169/**
170 * Use DMA SPI to read rate and acceleration data from the ADIS16470 IMU and
171 * return the robot's heading relative to a starting position and instant
172 * measurements
173 *
174 * The ADIS16470 gyro angle outputs track the robot's heading based on the
175 * starting position. As the robot rotates the new heading is computed by
176 * integrating the rate of rotation returned by the IMU. When the class is
177 * instantiated, a short calibration routine is performed where the IMU samples
178 * the gyros while at rest to determine the initial offset. This is subtracted
179 * from each sample to determine the heading.
180 *
181 * This class is for the ADIS16470 IMU connected via the primary SPI port
182 * available on the RoboRIO.
183 */
184

197 * @brief Customizable constructor. Allows the SPI port and CS to be

202 * GetRate()

242 * @brief Returns the current integrated angle for the axis specified.
243 *
244 * The angle is based on the current accumulator value corrected by
245 * offset calibration and built-in IMU calibration. The angle is continuous,
246 * that is it will continue from 360->361 degrees. This allows algorithms
247 * that wouldn't want to see a discontinuity in the gyro output as it sweeps
248 * from 360 to 0 on the second time around. The axis returned by this
249 * function is adjusted based on the configured yaw_axis.
250 *
251 * @return the current heading of the robot in degrees. This heading is based
252 * on integration of the returned rate from the gyro.
253 */
