133/**
134 * Given an 32 bit FPGA time, expand it to the nearest likely 64 bit FPGA time.
135 *
136 * Note: This is making the assumption that the timestamp being converted is
137 * always in the past. If you call this with a future timestamp, it probably
138 * will make it in the past. If you wait over 70 minutes between capturing the
139 * bottom 32 bits of the timestamp and expanding it, you will be off by
140 * multiples of 1<<32 microseconds.
141 *
142 * @param[in] unexpandedLower 32 bit FPGA time
143 * @param[out] status the error code, or 0 for success
144 * @return The current time in microseconds according to the FPGA (since FPGA
145 * reset) as a 64 bit number.
146 */
147uint64_t HAL_ExpandFPGATime(uint32_t unexpandedLower, int32_t* status);

149/**
150 * Call this to start up HAL. This is required for robot programs.
151 *
152 * This must be called before any other HAL functions. Failure to do so will
153 * result in undefined behavior, and likely segmentation faults. This means that
154 * any statically initialized variables in a program MUST call this function in
155 * their constructors if they want to use other HAL calls.
156 *
157 * The common parameters are 500 for timeout and 0 for mode.
158 *
159 * This function is safe to call from any thread, and as many times as you wish.
160 * It internally guards from any reentrancy.
161 *
162 * The applicable modes are:
163 * 0: Try to kill an existing HAL from another program, if not successful,
164 * error.
165 * 1: Force kill a HAL from another program.
166 * 2: Just warn if another hal exists and cannot be killed. Will likely result
167 * in undefined behavior.
168 *
169 * @param timeout the initialization timeout (ms)
170 * @param mode the initialization mode (see remarks)
171 * @return true if initialization was successful, otherwise false.
172 */
173HAL_Bool HAL_Initialize(int32_t timeout, int32_t mode);
