22/**
23 * The Command class is at the very core of the entire command framework.
24 *
25 * Every command can be started with a call to Start(). Once a command is
26 * started it will call Initialize(), and then will repeatedly call Execute()
27 * until the IsFinished() returns true. Once it does,End() will be called.
28 *
29 * However, if at any point while it is running Cancel() is called, then the
30 * command will be stopped and Interrupted() will be called.
31 *
32 * If a command uses a Subsystem, then it should specify that it does so by
33 * calling the Requires() method in its constructor. Note that a Command may
34 * have multiple requirements, and Requires() should be called for each one.
35 *
36 * If a command is running and a new command with shared requirements is
37 * started, then one of two things will happen. If the active command is
38 * interruptible, then Cancel() will be called and the command will be removed
39 * to make way for the new one. If the active command is not interruptible, the
40 * other one will not even be started, and the active one will continue
41 * functioning.
42 *
43 * This class is provided by the OldCommands VendorDep
44 * *
45 * @see CommandGroup
46 * @see Subsystem
47 */
