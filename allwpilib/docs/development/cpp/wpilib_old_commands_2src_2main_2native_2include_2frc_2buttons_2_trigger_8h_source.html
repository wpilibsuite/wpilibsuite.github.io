16/**
17 * This class provides an easy way to link commands to inputs.
18 *
19 * It is very easy to link a polled input to a command. For instance, you could
20 * link the trigger button of a joystick to a "score" command or an encoder
21 * reaching a particular value.
22 *
23 * It is encouraged that teams write a subclass of Trigger if they want to have
24 * something unusual (for instance, if they want to react to the user holding
25 * a button while the robot is reading a certain sensor input). For this, they
26 * only have to write the Trigger::Get() method to get the full functionality of
27 * the Trigger class.
28 */
