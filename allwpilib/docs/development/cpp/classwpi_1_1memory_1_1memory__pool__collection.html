inlinenoexcept

\effects Moving a memory_pool_collection object transfers ownership over the free lists, i.e. the moved from pool is completely empty and the new one has all its memory. That means that it is not allowed to call deallocate_node() on a moved-from allocator even when passing it memory that was previously allocated by this object.

inlinenoexcept
Returns
The amount of nodes available in the free list for nodes of given size as defined over the BucketDistribution. This is the number of nodes that can be allocated without the free list requesting more memory from the arena.
Note
Array allocations may lead to a growth even if the capacity_left is big enough.
