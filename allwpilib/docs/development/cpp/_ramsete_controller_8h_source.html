19/**
20 * Ramsete is a nonlinear time-varying feedback controller for unicycle models
21 * that drives the model to a desired pose along a two-dimensional trajectory.
22 * Why would we need a nonlinear control law in addition to the linear ones we
23 * have used so far like PID? If we use the original approach with PID
24 * controllers for left and right position and velocity states, the controllers
25 * only deal with the local pose. If the robot deviates from the path, there is
26 * no way for the controllers to correct and the robot may not reach the desired
27 * global pose. This is due to multiple endpoints existing for the robot which
28 * have the same encoder path arc lengths.
29 *
30 * Instead of using wheel path arc lengths (which are in the robot's local
31 * coordinate frame), nonlinear controllers like pure pursuit and Ramsete use
32 * global pose. The controller uses this extra information to guide a linear
33 * reference tracker like the PID controllers back in by adjusting the
34 * references of the PID controllers.
35 *
36 * The paper "Control of Wheeled Mobile Robots: An Experimental Overview"
37 * describes a nonlinear controller for a wheeled vehicle with unicycle-like
38 * kinematics; a global pose consisting of x, y, and theta; and a desired pose
39 * consisting of x_d, y_d, and theta_d. We call it Ramsete because that's the
40 * acronym for the title of the book it came from in Italian ("Robotica
41 * Articolata e Mobile per i SErvizi e le TEcnologie").
42 *
43 * See <https://file.tavsys.net/control/controls-engineering-in-frc.pdf> section
44 * on Ramsete unicycle controller for a derivation and analysis.
45 */
