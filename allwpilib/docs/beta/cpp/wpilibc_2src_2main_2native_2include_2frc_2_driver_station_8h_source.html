317 /**
318 * Wait until a new packet comes from the driver station.
319 *
320 * This blocks on a semaphore, so the waiting is efficient.
321 *
322 * This is a good way to delay processing until there is new driver station
323 * data to act on.
324 *
325 * Checks if new control data has arrived since the last waitForData call
326 * on the current thread. If new data has not arrived, returns immediately.
327 */
328 static void WaitForData();
329
330 /**
331 * Wait until a new packet comes from the driver station, or wait for a
332 * timeout.
333 *
334 * Checks if new control data has arrived since the last waitForData call
335 * on the current thread. If new data has not arrived, returns immediately.
336 *
337 * If the timeout is less then or equal to 0, wait indefinitely.
338 *
339 * Timeout is in milliseconds
340 *
341 * This blocks on a semaphore, so the waiting is efficient.
342 *
343 * This is a good way to delay processing until there is new driver station
344 * data to act on.
345 *
346 * @param timeout Timeout
347 *
348 * @return true if new data, otherwise false
349 */
350 static bool WaitForData(units::second_t timeout);
351
352 /**
353 * Return the approximate match time.
354 *
355 * The FMS does not send an official match time to the robots, but does send
356 * an approximate match time. The value will count down the time remaining in
357 * the current period (auto or teleop).
358 *
359 * Warning: This is not an official time (so it cannot be used to dispute ref
360 * calls or guarantee that a function will trigger before the match ends).
361 *
362 * The Practice Match function of the DS approximates the behavior seen on
363 * the field.
364 *
365 * @return Time remaining in current match period (auto or teleop)
366 */
367 static double GetMatchTime();
